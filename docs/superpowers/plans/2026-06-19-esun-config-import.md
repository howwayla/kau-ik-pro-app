# 玉山設定檔匯入實作計畫

## 背景

玉山證券 API 的申請與登入流程和富邦、台新不同。官方流程會給使用者 `config.ini` 或 `config.simulation.ini`，其中包含 API URL、API Key、API Secret 與帳號；另外仍需要 `.p12` 憑證檔、玉山登入密碼、憑證密碼。

因此玉山不適合讓一般使用者手動輸入 API Key / API Secret。更好的體驗是匯入設定檔，再補上兩個密碼與憑證檔。

## 決策

- 富邦、台新維持目前手動欄位流程。
- 玉山改為：
  - 匯入 `config.ini` / `config.simulation.ini`。
  - 選擇 `.p12` 憑證檔。
  - 輸入玉山登入密碼。
  - 輸入憑證密碼。
- 設定檔中的 API Key / API Secret / 帳號只進入既有安全儲存流程，不寫入一般設定檔。
- 若玉山設定檔中的憑證路徑是相對路徑，不自動採用，避免桌面 app 啟動位置不同造成登入失敗；使用者仍以「選擇憑證檔」指定實際檔案。

## 實作順序

1. 先補玉山設定檔解析與表單驗證測試。
2. 實作 `config.ini` 解析、必要欄位檢查、相對憑證路徑保護。
3. 將券商設定精靈的玉山畫面改成匯入設定檔流程。
4. 跑測試與桌面打包前檢查。
